# Password Policy

**Version:** 1.0  
**Last Updated:** June 2026  
**Owner:** Information Security Team  
**Review Date:** June 2027  
**Classification:** Internal  

---

## 1. Purpose

This policy defines the requirements for creating, managing, and protecting passwords across all organisation systems. Strong passwords are a fundamental control for protecting user accounts and sensitive data.

---

## 2. Scope

This policy applies to all employees, contractors, consultants, and third parties with access to organisation systems.

---

## 3. Password Requirements

### 3.1 Standard User Accounts

- Minimum length of 12 characters
- Must include at least one uppercase letter
- Must include at least one lowercase letter
- Must include at least one number
- Must include at least one special character
- Must not contain the user's name or username
- Must not be a previously used password

### 3.2 Privileged Accounts

- Minimum length of 16 characters
- Must meet all standard password requirements above
- Must be changed every 90 days
- Must be unique and not reused across systems

### 3.3 Service Accounts

- Minimum length of 20 characters
- Must be randomly generated
- Must be stored in an approved password manager
- Must be rotated at least annually

---

## 4. Password Management

**4.1 Password Storage**
- Passwords must never be written down or stored in plain text
- Passwords must be stored in an organisation approved password manager
- Passwords must never be stored in spreadsheets, notes apps, or email

**4.2 Password Sharing**
- Passwords must never be shared with anyone including IT staff
- IT staff will never ask for a user password
- If a password must be shared for a technical reason it must be changed immediately after

**4.3 Default Passwords**
- Default passwords on all systems and devices must be changed immediately upon setup
- No system may go live with a default password in place

---

## 5. Password Reset Process

- Users can reset their own password through the self service portal
- Identity must be verified before a password reset is completed
- Temporary passwords must be changed on first login
- Password reset requests via phone or email must be verified through a secondary channel

---

## 6. Account Lockout

- Accounts are locked after 5 consecutive failed login attempts
- Locked accounts require IT team intervention or self service unlock via verified identity
- Account lockout events are logged and reviewed

---

## 7. Multi-Factor Authentication

- MFA is mandatory for all accounts accessing sensitive systems
- MFA is mandatory for all remote access
- MFA is mandatory for all privileged accounts
- Approved MFA methods are authenticator apps and hardware tokens
- SMS based MFA is permitted only where no other option is available

---

## 8. Password Auditing

- Password strength is audited quarterly
- Compromised passwords are identified using approved breach detection tools
- Users with compromised passwords are notified and required to change immediately
